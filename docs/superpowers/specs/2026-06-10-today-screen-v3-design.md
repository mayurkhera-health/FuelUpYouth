# Today Screen v3 — Design Spec

**Date:** 2026-06-10  
**Status:** Approved  
**Replaces:** Current Today.jsx (Greeting + CountdownHero + FuelScoreCard + StreakCard + WeekBarChart + NutrientsReportCard + TomorrowAlert + NextMealsStrip + HydrationTracker zones)

---

## What This Screen Is

The Today screen is a daily briefing, not a dashboard. Three zones answer three questions in sequence:
- **Zone A — Broadcast Card:** "You are the athlete — here is your identity and status"
- **Zone B — Daily Mission:** "Here is exactly what to do today, in order"
- **Zone C — Science Edge:** "Here is why it matters — explained as performance, not nutrition"

Designed for a 15-year-old athlete who does not want to track macros. Nutrition feels like preparation for competition.

---

## Design Decisions

- **Missing athlete fields** (`last_name`, `jersey_number`, `team_name`): gracefully degrade — show what exists, omit silently what doesn't
- **Missing event fields** (`opponent`): gracefully degrade
- **Bottom nav:** keep existing AppShell top tab bar unchanged
- **Mission item tap → API:** POST to existing `/api/meal-logs/` endpoint; stub `confirm` endpoint later
- **Science fix tap:** toast only ("→ Added to meal plan"), no API call until endpoint exists
- **Design tokens:** FuelUp green palette (`#2d6a4f`, `#1b3a2a`, `#52b788`), Nunito + DM Sans fonts, existing border/surface tokens

## Architecture

**`api/services/today_service.py`** — add two functions:

1. `calculate_performance_forecast(traffic_light)` → `{sprint_capacity, energy_reserves, second_half_power, mental_focus}` (pure math, no Claude)

## Component Specs

### BroadcastCard

- **LiveTicker strip** (34px): pulsing live dot, event string, live countdown `H:MM TO KICKOFF`. Updates every 30s. Turns red < 30 min. Shows "IN PROGRESS" when event passed.
- **AthleteIdentity row**: left = position line + `FIRST_NAME / LAST_NAME` (omit last_name if missing) + team/city; right = SVG readiness dial (r=29, stroke-dashoffset animates from 182→final on mount, 0.9s cubic-bezier). Score number counts 0→final at +2/14ms.
- **PerformanceTagline bar**: solid `#2d6a4f` fill (only solid accent on page). Emoji + tagline text + arrow. Navigates to meal-plan tab on tap. Text and emoji vary by score threshold (≥85/≥70/≥50/<50).
- **StatsRow**: 4-cell grid — Fuel Score, Carbs %, Iron %, Kickoff countdown. Colors match score thresholds (green/amber/red).

### PerformanceForecast

- 2×2 grid of metric bars
- All 4 calculated in Python by `calculate_performance_forecast()`:
  - Sprint Capacity: carbs×0.40 + iron×0.35 + protein×0.25
  - Energy Reserves: calories×0.60 + carbs×0.40
  - Second-Half Power: iron×0.50 + carbs×0.30 + water×0.20
  - Mental Focus: calories×0.40 + water×0.35 + protein×0.25
- Bars animate 0→final on mount, 0.7s ease
- Color thresholds: ≥75 green, ≥50 amber, <50 red

### DailyMission

- Header: event type + date eyebrow, "Today's Mission" title, X/5 counter, 64px progress bar
- 5 MissionItem rows — always 5, never empty (rest day has its own 5 items)
- Item states: PENDING, URGENT (amber), CRITICAL (red), DONE (green ✓)
- Tap to toggle DONE: strikethrough label, opacity 55%, counter increments, toast fires, POST to `/api/meal-logs/`
- Un-tap: reverts all, no API call
- Mission items generated by Python `get_mission_items()` — varies by event type (game/practice/strength/pregame_day/rest)

### ScienceEdge

- 3 rotating cards, auto-advances every 8s, dot indicators clickable
- Card priority: iron < 50% → Card 1 (Iron) always first; otherwise day-of-week rotation
- 3 fixed cards: Iron · Carb Loading Timing · Calcium/Bone Mass
- Food fix rows: tap → toast "→ [food] added to meal plan" (no API call yet)
- Source line at bottom of each card

### QuickRow

- Left: `HydrationMini` — value in oz, 10 cup icons (filled/empty), tap fills/unfills optimistically, POST `/api/water-log/`
- Right: `CaloriesQuick` — logged vs target, %, 2px progress bar, "X kcal remaining"

### Toast

- Fixed, bottom 80px, above bottom nav
- Solid `#2d6a4f` background, dark text
- 0.22s show/hide transition, auto-dismiss 2.2s
- Fires on: mission completion, science fix tap, log button tap

---

## Animations — only these three

1. Readiness dial: stroke-dashoffset 182→final, 0.9s cubic-bezier(.4,0,.2,1), +200ms delay. Score counter +2/14ms.
2. Progress bars (forecast + calories): 0→final, 0.7s ease, on mount
3. Live dot + NOW tag pulse: opacity 1→0.4→1, 1.4s infinite, CSS only

No slide-ins, no bounces, no fade-ins.

## Critical Rules

1. Python calculates all numbers. Claude never calculates on this screen.
2. PerformanceTagline is the only solid accent element. Everything else uses accent at low opacity.
3. Progress bars are always 2px height.
4. Iron is always flagged for girls when below 50% — overrides all other priority logic.
5. Countdown updates every 30s, not every second.
6. Mission list is never empty — rest day has 5 rest-day items.
7. Science Edge leads with performance outcome before mechanism.
8. Log button is always visible but never the primary CTA.
9. Legal disclaimer appears at bottom.
